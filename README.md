# 🧠 Retos de Lógica — Del Diagrama al Código

> **Aprende a pensar antes de escribir código.**

Este repositorio es una colección de ejercicios de lógica de programación organizados por niveles de dificultad. La filosofía es simple: **primero piensa, luego dibuja, luego programa.**

---

## 🎯 ¿Para quién es esto?

- Personas que están aprendiendo a programar
- Estudiantes que quieren fortalecer su pensamiento lógico
- Docentes que buscan ejercicios prácticos para sus clases
- Cualquiera que quiera mejorar su capacidad de resolver problemas

---

## 📐 La Metodología: Diagrama → Código

Cada reto tiene **dos fases**:

| Fase | Descripción |
|------|-------------|
| 📐 **Fase 1 — Diagrama de Flujo** | Dibuja la solución usando símbolos de diagrama de flujo. No escribas código todavía. |
| 💻 **Fase 2 — Código** | Una vez que el diagrama está listo y funciona, tradúcelo al lenguaje que prefieras. |

> 💡 Esta metodología imita cómo trabajan los ingenieros de software reales: diseñan antes de implementar.

## 🚀 ¿Cómo empezar?

1. Lee [`COMO_RESOLVER.md`](./COMO_RESOLVER.md) — aprende la metodología y cómo hacer diagramas
2. Revisa [`HERRAMIENTAS.md`](./HERRAMIENTAS.md) — elige tu herramienta de diagramación
3. Empieza por el **Nivel 1** aunque te sientas avanzado — los primeros retos son engañosamente simples
4. Para cada reto: **lee → diagrama → prueba el diagrama → codifica**

## 📜 Reglas del Repositorio

1. **No copies soluciones de internet**. El objetivo es pensar, no buscar.
2. **El diagrama va primero**. Siempre.
3. **Cualquier lenguaje es válido** en la Fase 2: Python, JavaScript, Java, C++, Pseudocódigo...
4. **Comparte tu solución** con otros y compara approachs.
